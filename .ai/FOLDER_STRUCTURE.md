# ADMIN PLACEMENT RULES

? New admin page
  вҶ’ src/app/(dashboard)/{route}/page.tsx
  вҶ’ src/app/(dashboard)/{route}/loading.tsx
  вҶ’ Add role check in middleware.ts OR useRoleGuard in page

? New DataTable page (most common pattern)
  вҶ’ page.tsx uses DataTable from "@computer-store/ui"
  вҶ’ service function in src/services/{resource}.service.ts
  вҶ’ types in src/types/{resource}.types.ts
  вҶ’ column definitions inline in page (unless reused elsewhere)

? New form (ProductForm, ImportForm, etc.)
  вҶ’ src/components/{domain}/{ResourceName}Form.tsx
  вҶ’ Zod schema вҶ’ src/lib/validators.ts
  вҶ’ Uses react-hook-form + zod resolver

? New chart
  вҶ’ src/components/dashboard/{ChartName}.tsx
  вҶ’ Use Recharts (never D3 or Chart.js)

? New role-restricted action
  вҶ’ Check role in middleware.ts (route-level)
  вҶ’ Or: useRoleGuard hook (component-level)
  вҶ’ Never rely on UI hiding alone вҖ” backend enforces roles too
